I created a poll last week asking the community which coin I should write review next. RaiBlocks won with 40% of the vote. I‚Äôve since spent several hours researching RaiBlocks and compiling everything I‚Äôve learned in this article. I hope you find it helpful.

As Bitcoin‚Äôs price rose over 2017, so did fees required to use the network. Bitcoin Core developers have been against a block-size increase and are promising that the Lightning network will solve the fee problem. However, there are many uncertainties with the Lightning network including when it will be released and mass adopted.

This Bitcoin fee fiasco has generated significant interest in other cryptocurrencies with low fees and fast transaction times. RaiBlocks just so happens to be a prime candidate. As Bitcoin‚Äôs market cap dominance dropped from 60% to 33%, alternative cryptocurrencies saw large increases in marketcap and RaiBlocks was one of them. From a price under $0.20 at the start of December, RaiBlocks‚Äô price skyrocketed to almost $35 at its peak and is now stabilizing at around $22.

One reason for this rise is the claim that RaiBlocks can support near instant and free transactions with unlimited throughput capacity while being sufficiently decentralized. This, of course, raises many people‚Äôs eyebrows since the best property of cryptocurrencies, decentralization, has always meant shitty scalability.

Let‚Äôs dive into RaiBlocks to see what it is, how it works, and whether it‚Äôs actually decentralized. We‚Äôll also look at ecosystem factors like it‚Äôs team, investors, etc.

10,000 feet summary

  • Started in 2014. The main net was launched in 2017 and it‚Äôs fully functional.
  • Fixed, maximum supply of 133,248,289 RaiBlocks. All RaiBlocks has been released.
  • 0 transaction fee, unlimited throughput capacity, > 5 second confirmation time.
  • Uses a block-lattice data structure instead of the blockchain.
  • Consensus is maintained through a voting mechanism. Votes = # of raiblocks owned and delegated.
  • RaiBlocks didn‚Äôt conduct an ICO. RaiBlocks was distributed through a faucet. 7,000,000 RaiBlocks retained for dev funds.

Is RaiBlocks a scam?

I‚Äôve scoured through the whitepaper and looked over the team‚Äôs history and believe that RaiBlocks is everything it claims, with a few caveats. There are many open questions around RaiBlocks‚Äôs consensus algorithm. It‚Äôs not well tested and to my understanding, lacks peer review.

Although RaiBlocks in its current state does indeed support near instant and free transactions with unlimited throughput capacity, the jury is still out on whether the consensus algorithm is correct and robust.

How does RaiBlocks work?

Unlike most cryptocurrencies that use a traditional blockchain datastructure, RaiBlocks uses a novel data structure known as a block-lattice. While the blockchain consists of a singular global chain of blocks, every RaiBlocks account has its own blockchain.

The only account that can publish a transaction to an account‚Äôs blockchain is the account itself.

To send funds, the sender needs to issue a ‚Äúsend‚Äù transaction in its own blockchain, and the receiver needs to issue a corresponding ‚Äúreceive‚Äù transaction in its own blockchain. This is why it‚Äôs so fast. An account can very quickly update its own blockchain. These transactions are propagated across the network and as long as no validator discovers a conflict, the transaction is, for all intents and purposes, considered confirmed.

The RaiBlocks network consists of wallets and validator nodes. Wallets send transactions to a validator node, and the node first validates the transactions by checking them against their internal state of the network. The node then passes along the transaction to all other validator nodes its connected to. Every subsequent node performs the same sequence of action until the whole network has seen the transactions.

A validator node keeps track of the state of the entire network by storing the blockchains of every single RaiBlocks address. Conflicts or ‚Äúforks‚Äù happen when a published transaction conflicts with the internal state of a validator node. To resolve the conflict, the node initiates a voting process by announcing the conflict to other validator nodes. The voting process occurs over a predefined period where each node votes on a state of the network it believes is correct. The number of votes a node has equals the amount of RaiBlocks it owns + votes delegated from other accounts. When a state has more than 133,248,289/2 votes (133,248,289 is the total number of XRBs in existence), it‚Äôs automatically accepted as the real state of the network.

The network provides no rewards for running a RaiBlocks validator.

To mitigate spam, an account needs to perform easy proof of work to add a transaction to its own blockchain. This is, of course, not a cure all as transactions can still be created within a few seconds.

RaiBlocks team and history

The key developer behind RaiBlocks is Colin LeMahieu. LeMahieu started developing RaiBlocks in 2014 and the network was released in 2017. RaiBlocks didn‚Äôt hold an ICO and instead distributed the cryptocurrency through a faucet on a website. Users need only complete a captcha in order to receive RaiBlocks.

The team behind RaiBlocks is small and fairly unknown but with the cryptocurrency‚Äôs recent success, LeMahieu is in the process of building a larger development team in order to cultivate a stronger ecosystem around RaiBlocks.

PayPal lead engineer + RaiBlocks

It‚Äôs interesting to note that a lead PayPal engineer built a checkout app for RaiBlocks as a weekend hack project and wrote about it on Medium. For a lead PayPal engineer to be interested in RaiBlocks to actually built something on it, even if it was just a ‚Äúweekend hack project‚Äù, is quite significant for this nascent cryptocurrency.

RaiBlocks vs IOTA

Another cryptocurrency that also offers near instant and free transactions with high scalability is IOTA. IOTA has been extremely ‚Äúhot‚Äù recently with prices skyrocketing from under a $1 to more than $5. It‚Äôs not surprising that IOTA is frequently compared to RaiBlocks. Here‚Äôs how the two cryptocurrencies differ.

IOTA, like RaiBlocks, offers near instant and free transactions. However, the two systems are very different. IOTA uses a DAG data structure (called Tangle) where every node in the graph represents one whole transaction. In order to issue a new transaction, an account needs to validate two previous transactions through proof of work. The new transaction would then ‚Äúreference‚Äù or connect to the two previous transactions in the Tangle.

In IOTA, transactions happen in a single phase and is stored in the global Tangle. In RaiBlocks, transactions happen in two parts: a ‚Äúsend‚Äù transaction in the sender‚Äôs blockchain and a ‚Äúreceive‚Äù transaction in the receiver‚Äôs blockchain.

One major advantage RaiBlocks has over IOTA is in decentralization. IOTA currently needs a centralized Coordinator to ensure network security and direct network progress. This is because IOTA‚Äôs network security is directly tied to the amount of transactions occuring in the network. In low transaction conditions, transactions take longer to confirm and the network becomes more vulnerable to attacks. In theory, the Coordinator will eventually become unnecessary as the network reaches a critical mass of adoption but IOTA is far from this reality. A scary consequence of the Coordinator is the fact that IOTA‚Äôs developers can, and has, frozen user funds at a whim (source).

The security of RaiBlocks‚Äôs network, on the other hand, does not depend on the amount of transactions going through it. As such, RaiBlocks currently doesn‚Äôt need a centralized entity as a crutch to be secure.

IOTA has an ecosystem advantage over RaiBlocks. IOTA has more users and the IOTA Foundation has more business connections. For example, the Foundation recently launched an IOTA Data Marketplace initiative with many large companies participating.

Weaknesses

Questions around the consensus algorithm

As mentioned above, there are many open questions around the consensus algorithm. Given that RaiBlocks is so young, it is not well tested and lacks peer review. The risk is definitely real that there is a fatal flaw in the consensus algorithm. For example, what happens if there are not enough votes for a quorum in order to resolve a network conflict? Currently validator nodes rely on the maximum supply number (133,248,289) to determine a quorum and there definitely could be instances where not enough validator nodes participate in the voting process.

Another big question is in the event the RaiBlocks network gets separated for a long period of time (perhaps due to network issues or other freak occurences), what happens when you rejoin the two networks? Will the network be crippled from all the voting processes that would inevitably take place? Is there a mathematical guarantee that the network will recover?

Spam vulnerability

RaiBlocks, being inherently feeless, is highly vulnerable to spam. I could run a script that simply has multiple accounts send transactions to each other continuously creating massive blockchains that could be a problem for validators. It costs almost nothing to create spam in the system yet can be incredibly costly. The storage costs of blockchain cryptocurrencies is predicatable based on block generation times and maximum block sizes. This is not the case with RaiBlocks.

Expect these spam attacks to occur as RaiBlocks increases in popularity and we will have to see how the development responds.

RaiBlocks doesn‚Äôt incentivize decentralization

RaiBlocks validator nodes are quite expensive to run with high network throughput costs and modest processing and storage costs. Since there‚Äôs no reward for running a RaiBlocks validator node, validator node operators are prone to centralization.

Some people argue that businesses relying on the RaiBlocks network will be incentivized to run validator nodes to secure the network. Large investors might also be incentivized run validator nodes. However, we have yet to see if this is true.

Limited applications - no smart contracts

It‚Äôs becoming increasingly clear how powerful smart contracts are and I have a strong feeling that the winning cryptocurrencies in the future must support smart contracts. RaiBlocks‚Äôs system is so novel and different that I wonder if smart contracts are even possible for RaiBlocks?

Bottomline

RaiBlocks works as described - near instant and feeless transactions with unlimited transaction throughput. This cryptocurrency has a lot of promise and can be a serious challenge to big names like IOTA and even Bitcoin. However, RaiBlocks is currently being held back by an unproven consensus algorithm, spam vulnerability, and the lack of potential for smart contracts.
